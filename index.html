RoboMME-Interference

Benchmarking Robot Memory Under Interference

A cross-session benchmark for memory-augmented VLAs

Soumil Shah

Built on RoboMME-style robot memory tasks

Paper Code Data Protocol Results

Abstract

Robots deployed in realistic settings accumulate experience across many sessions, tasks, and users. Current robot-memory evaluations often focus on short or within-episode histories. We introduce RoboMME-Interference, a cross-session interference protocol over RoboMME task families that measures whether a policy can still use a relevant prior session after unrelated robot experiences are inserted into the history buffer.

We evaluate nine runnable memory-policy variants across nine task families and 18,450 rollouts. Perceptual memory systems improve success when the relevant session is nearby, but the benefit decays as distractor sessions increase. The benchmark makes interference sensitivity directly measurable for memory-augmented robot policies.

Key Findings

  • Nearby relevant memory helps. FrameSamp-Modul rises from 18.2% with no history to 45.3% at k0.
  • Interference erodes the gain. FrameSamp-Modul drops from 45.3% at k0 to 19.3% at k7.
  • Memory mechanisms behave differently. Perceptual variants separate clearly under interference; recurrent TTT variants remain mostly flat in this grid.

Protocol

Each query episode is evaluated with the same policy interface and a different external history buffer. The relevant lesson always appears before any distractor sessions. Distractors come from different task families to avoid constructing arbitrary contradictory priors within a single family.

What This Benchmark Is

  • A cross-session interference protocol over RoboMME-style memory tasks.
  • A measurement of how visual-history memory systems behave as unrelated sessions accumulate.
  • A released result grid with code, data, generated figures, and analysis scripts.

What This Benchmark Is Not

  • It is not a new simulator benchmark independent of RoboMME.
  • It is not a claim that current robot-memory systems solve long-term memory.
  • It is not an oracle-symbolic memory benchmark; headline systems use the visual-history pathway.
